vfs - прослойка для между файловой системой и операционной системой (ядром). Служит для общения (по типу API) програм с железом (по средствам системных вызовов).
tmpfs - виртуальная ФС для виртуальной ФС.

/dev/ - файлы в данном каталоге отвечают за взаимодействие с реальными устройствами (через драйверы)
существует два типа устройств (первый столбец из ls -l):
С - символьное (characters) считывание\посылание символов (терминал, принтер)
B - блочное (blocks) считывание\посылание блоков данных (диски), минимальная единица - сектор.

Вывести какие используются файловые системы
df -hT
Посмотреть информацию о каталоге
ls -ld /
Вывести в список inodes 
ls -li

ls -l /dev   //в вывод добавляются major (в рамках драйвера) и minor (личная модификация) numbers устройства (cat /proc/devices или cat /proc/partitions  -- список драйверов, major numbers и minor )


Вывести информацию о файле\каталоге
stat /


Создание нового устройства
mknod /dev/sda10 b 8, 1      // 8 major num; 1 minor num


/sys/ - совокупность данных о работе ОС (шины, устойства, модули, модули ядра lsmod (/sys/module/))
Перечитать данные по шине, например подключили новый диск или изменили его размер (scsi), что бы не перезагружать ОС
echo "- - -" > /sys/class/scsi_host/host0/scan
или
ls -l /sys/dev/block   //идентификация устройства, его путь в /sys/devices/pci...
echo 1 > /sys/devices/pci.../.../rescan   //запустится перечтение

/sys/dev/ - интерфейс драйвера


/proc/ - текущие метрики системы (по процессам)

Посмотреть куда процессы пишут логи
/proc/_pid_/fd    //fd - file descriptor (файловый дескриптор)
stdin - 0
stdout - 1
stderr - 2

Последний системный вызов процесса (обращение к ядру)
/proc/_pid_/wchan

Инфа по памяти, подробная
/proc/meminfo  // более краткая, исп. утилиту free

аналог lsof
ls -l /proc/*/fd 2>/dev/null 

восстановить удалённый файл, если он ещё открыт в программе (находится в ОЗУ)
cp /proc/_pid_/fd/_номер_потока_   // будет помечен как (deleted)

Очистить место на диске после удаления большого файла, при учловии что он ещё открыт в программе (находится в ОЗУ)
> /proc/_pid_/fd/_номер_потока_   // будет помечен как (deleted), в него запишется "ничего" и место освободится, хотя файл ещё находится в ОЗУ

Можно скорректировать работы oom для процесса, внеся данные в
/proc/_pid_/oom_score_adj
Скрипт что бы посмотреть по системе, у какого процесса какой текущий oom_score
fot i in $(ls -l /proc/*/oom_score | awk '{print $NF}' ); do echo -n "$i " ; cat $i; done

Сбросить ОЗУ cashe (кэш) память
3 > /proc/sys/vm/drop_cashes

Настойки параметров ядра ()
sysctl -a   =   /proc/sys/vm/...    // из sysctl -a . заменяем на / для ls -l /proc/sys/vm/...   Конфиг лежит в /etc/sysctl.conf
