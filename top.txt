top ([1]-кол-во ядер, [shift]+[>],[<]-сортировка, [shift]+[m]-меню(сортировка), )
LA - среднее количество процессов в очереди на ожидание ресурсов (1, 5, 15). ядра проц. >= LA. Процессы ждут: либо процессор (la(1)=us+sy+ni), либо устройство ввода\вывода (wa)
us - user
sy - system (kernel)
ni - nice (удобно задать напр. nginx пониженный и отслеживать его)
id - idle
wa - wait (i\o)
hi - hardware iteraction (сбоит железо)
si - systen iteraction
st - stolen time (for linux virtualization)

virt - запрошенная у ядра память
res - фактически испозьзуемая пямять
S - статус процесса: 
D (disk) - невоспринимает никакие сигналы, ожидает ответ от диска (ввода\вывода). 
R (Running), выполняется
S (sleeping), ожидает (события)
T (terminate), остановлен
Z (zombie), процесс зомби


Для управления Top используются горячие клавиши. Вот некоторые из них:
Shift+F — открыть меню для редактирования колонок таблицы.
Shift+P — сортировка по использованию ЦПУ.
Shift+M — сортировка по использованию ОЗУ.
Shift+N — сортировка по номеру PID.
Shift+T — сортировка по uptime.
Shift+U — сортировка по имени пользователя или выбор процессов, принадлежащих конкретному пользователю (необходимо ввести имя пользователя).
n — отображать только определенное число процессов (нужно ввести количество процессов).
k — послать сигнал kill процессу (необходимо ввести PID процесса).
r — изменить приоритет процесса.
1 — отобразить статистику по всем ядрам процессора.
Shift+H — показать потоки.
Shift+V — древовидный вид процессов в таблице.
q — выйти из программы.
h — показать горячие клавиши.
